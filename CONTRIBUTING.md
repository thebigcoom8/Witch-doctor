# Contributing to Witch Doctor

## Welcome & Ways To Contribute

Hey, thanks for being here. If you're thinking you need some kind of qualification to help — you don't. You, yes you reading this right now, already have everything it takes.

If you can code that's amazing — this is a big project and there's plenty to build. But if you can't that's completely fine, there's still a lot you can do. If you have clean reliable data, knowledge in harm reduction, experience with AI, or just spotted something that doesn't look right — there's a place for you here.

No experience required. No gatekeeping. Just people trying to make sure others stay safe.

---

## Suggesting a Data Source

If you own or maintain a website or database covering anything related to our current or planned topics, we'd love to hear from you. Open an issue with the following:

- A link to the source
- A contact for whoever owns or maintains it — if that's not you, make sure you have their permission before submitting
- A brief note on what it covers and why you think it's a good fit

We review every submission personally. If it's reliable, well maintained, and fits the mission it'll go on the list.

If you'd allow us to credit you we'll name drop you in the data source entry — both so you get proper thanks and so anyone using Witch Doctor knows exactly where the information came from. Just let us know in the issue whether you'd like to be credited and how.

---

## Reporting Wrong or Outdated Info

Every piece of information in Witch Doctor has a date showing when it was acquired. If something looks out of date, incorrect, or just doesn't seem right — please open an issue. Bad information in a harm reduction tool is dangerous and we'd rather hear about it a hundred times than miss it once.

If the info is broken and you know how to fix it, feel free to fork the repo, make your changes on a new branch and open a pull request. Just make sure to include your source so we can verify it before merging.

---

## Contributing Code

If you've got a fix, an improvement, or you want to take a crack at something on the [roadmap](https://github.com/thebigcoom8/Witch-doctor/blob/main/ROADMAP.md) — go for it. Fork the repo, build it on a new branch and open a pull request. Same rules as data contributions — include your sources and reasoning so we can review it properly before merging.

If you used AI to help write any of the code please credit it in your pull request — which tool you used and roughly how much of the code it wrote or contributed to. No judgment either way, we just believe in being transparent about how things are made.

If you want to add something that isn't on the [roadmap](https://github.com/thebigcoom8/Witch-doctor/blob/main/ROADMAP.md) go ahead and build it — just know it may not be accepted if it doesn't fit the mission. If you want to check before you put the work in open an issue first and make your case, we'll let you know where we stand.

And seriously, thank you. Every contribution helps make this more useful for the people who need it most.

---

## Contributing Data

If you have raw data you'd like to contribute you've got two options:

**Option 1 — Do it yourself:** Clean the data, add it on a new branch in the correct format with your sources clearly marked and dated, and open a pull request.

**Option 2 — Send it over as is:** Open an issue with the raw data attached. If you don't have the time or ability to clean it, we or another contributor will take care of it.

If you own or run a website or forum covering any of our topics, same two options apply. Either set up the scraper yourself the way you'd like it to work and open a pull request, or open an issue with the link, your contact info, and instructions for how you'd like us to scrape it.

If you know of a website or forum that would be a good source but you're not the owner, still open an issue with the link — just include contact info for the owner or proof that you've already reached out and gotten the go ahead. We'll always check with the owner before scraping anything we don't have explicit permission for.

## Code of Conduct

Witch Doctor is a space where everyone is welcome. Full stop.

If you make someone feel unsafe, unwelcome, or uncomfortable you'll be spoken to. If it continues your access to the repository will be revoked. We care more about maintaining a safe and welcoming community than we do about keeping any individual contributor around.

The same goes for anything that goes against the mission of this project. We'll have the conversation twice — but after that we'll just remove you. No drama, no lengthy debates, just gone.
